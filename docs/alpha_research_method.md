# Phase 2 method note — RAMP

## Goal

Phase 2 shifts the project from **execution research** to **alpha researcher style work**.

Instead of asking *when should I execute a parent order?*, the new question is:

> How should I combine fast and slow microstructure signals into a cost-aware market-neutral portfolio that still performs after turnover costs?

## Benchmark design

The alpha benchmark is deliberately structured to resemble a small research loop one might show in an interview:

1. calibrate a feature pool from **real LOB data**;
2. build a **multi-symbol synthetic universe** from that pool;
3. train several alpha models;
4. transform alpha scores into a **portfolio**;
5. compare strategies in **net bps** after explicit costs.

## Synthetic universe

Each pseudo-symbol is generated by stitching together blocks sampled from real AAPL order-book features.
This preserves realistic empirical relationships between:

- spread;
- imbalance;
- microprice deviation;
- signed flow;
- short-horizon returns.

A hidden return process is then added with:

- **regime dependence**;
- **multi-horizon structure**;
- **common-factor noise**;
- **idiosyncratic noise**;
- an explicit **trading-cost proxy**.

The realized return used in the portfolio backtest is a mixture of fast and slow latent returns.
So the best strategy is not a single-horizon predictor, but a method that can adapt its horizon emphasis.

## Baselines

All strategies use the same portfolio wrapper.
Only the alpha score changes.

- `imbalance`: cross-sectional ranking on L1 imbalance;
- `momentum`: ranking on slower return proxy;
- `ridge5`: a single-horizon ridge alpha;
- `uniform`: simple average of multi-horizon ridge predictions;
- `RAMP`: our regime-aware, cost-aware horizon-mixing model.

## RAMP

### Step 1 — Fit multiple horizon models

For each horizon

\[
H \in \{1,2,5,10,20\},
\]

we fit a ridge predictor

\[
\hat r_{t}^{(H)} = x_t^\top \beta_H.
\]

### Step 2 — Estimate regime reliability

On the validation set, for each observed regime `r` and each horizon `H`, we compute a positive reliability weight

\[
\rho_{r,H} = \max\big(\mathrm{corr}(\hat r_t^{(H)}, r_t^{\mathrm{realized}}), 0\big).
\]

These weights tell us which horizons are reliable in which regimes.

### Step 3 — Build fast and slow stacks

We define:

- fast horizons: \(\{1,2,5\}\)
- slow horizons: \(\{10,20\}\)

and aggregate them as

\[
s_t^{\text{fast}} = \sum_{H \in \mathcal H_f} w_{r,H}^{\text{fast}} \, \hat r_t^{(H)},
\]

\[
s_t^{\text{slow}} = \sum_{H \in \mathcal H_s} w_{r,H}^{\text{slow}} \, \hat r_t^{(H)},
\]

where the weights are normalized versions of the positive validation reliabilities.

### Step 4 — Regime-aware gating

An observable gate based on microstructure proxies mixes the two stacks:

\[
g_t = \sigma\big(a_1 \cdot \text{micro\_dev}_t - a_2 \cdot \text{ret}_{30,t} - a_3 \cdot \text{spread}_t\big),
\]

\[
s_t = g_t \, s_t^{\text{fast}} + (1-g_t) \, s_t^{\text{slow}}.
\]

### Step 5 — Temporal smoothing

To reduce churn, we smooth the score through time:

\[
\tilde s_t = \gamma \tilde s_{t-1} + (1-\gamma) s_t.
\]

### Step 6 — Cost-aware portfolio projection

We then shrink by risk and trading cost and form a market-neutral portfolio:

\[
z_{i,t} = \frac{\tilde s_{i,t}}{\widehat\sigma_{i,t} + c_0} \cdot \frac{1}{1 + \eta\,\text{cost}_{i,t}},
\]

center cross-sectionally,
apply a bounded nonlinearity, and normalize to unit gross exposure.

## Metrics

The backtest tracks:

- gross bps;
- net bps;
- hit rate;
- turnover;
- a simple Sharpe-like ratio.

The main metric is **mean net bps per step**.

## Why this is a better alpha-research project

Compared with the execution-only project, this phase shows that you can:

- design an alpha benchmark;
- think in terms of **cross-sectional portfolios**;
- account for **transaction costs**;
- use **validation-based model selection**;
- defend a custom method with a clear reason for why it should outperform.

That is much closer to the shape of an actual alpha-research conversation.
